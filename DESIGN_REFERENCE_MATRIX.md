# 设计参考矩阵

本文件记录已完成页面的信息架构参考边界。本轮完整性修复不继续扩展视觉设计。

| 本项目页面 | 主要参考 | 借鉴的交互 | 明确拒绝的模式 | 已落地表现 |
|---|---|---|---|---|
| 今日机会 | Plane；Twenty | 紧凑队列、优先级、列表与详情联动 | KPI Dashboard、项目管理术语 | 左侧机会队列，右侧客户上下文与行动按钮 |
| 内容作战台 | 组合式业务工作区 | 左侧上下文、中间编辑、右侧事实与风险 | 单一产品复制、每项能力独立卡片 | 三段式连续工作区与逐句定位 |
| 跟进与记忆 | Chatwoot；Twenty | 会话列表、沟通正文、客户属性与活动记录 | 客服工单术语、装饰性卡片时间线 | 会话工作区、客户时间线、记忆与预约 |
| 门店审核 | Chatwoot | 处理队列、完整上下文、明确下一步 | 总览 Dashboard 代替审核 | 完整正文、CTA、逐句事实与风险处理 |
| 活动与批量任务 | Plane | 紧凑明细、状态、失败原因、重试 | Sprint / Issue / Cycle | 顾问与渠道级任务明细和真实操作 |
| 顾问管理 | Twenty | 属性层级、列表与详情关系 | 完整 CRM 导航和销售漏斗 | 顾问列表、画像详情、保存与错误状态 |
| 设置 | shadcn/ui 交互原则 | 表单反馈、Toast、focus、disabled | 通用 shadcn Dashboard | 服务诊断、独立检查、导出和重置 |
| 试驾预约 | Cal.com / Cal.diy | 时间、参与人、地点、确认状态 | 点击后写死预约记录 | 可编辑时间、物品、备注和实际负责顾问 |
