# 设计参考矩阵

参考仅用于交互与信息架构，不复制品牌、代码、配色或完整导航。

| 本项目页面 | 主要参考 | 借鉴的交互 | 明确拒绝的模式 | 本次具体修改 |
|---|---|---|---|---|
| 今日机会 | Plane + Twenty | 紧凑待处理队列、优先级、列表与详情 | KPI Dashboard、项目管理术语 | 机会按状态和理由排列，详情展示客户上下文和推荐动作 |
| 内容作战台 | 组合研究 | 左上下文、中编辑、右事实/合规固定三栏 | 单一卡片堆叠、复制任一产品 | 编辑后立即使核验失效，右栏完成定位和重新核验 |
| 跟进与记忆 | Chatwoot + Twenty | 会话工作区、联系人上下文、活动记录 | 客服工单术语、卡片式假时间线 | 客户消息、顾问动作、预约和记忆处于同一连续过程 |
| 门店审核 | Chatwoot | 处理队列、完整上下文、下一步动作 | Dashboard 代替审核 | 完整正文与逐句证据并列，经理修改后必须重新核验 |
| 活动与批量任务 | Plane | 紧凑任务表、状态、失败原因、重试 | Sprint/Issue/Cycle 术语 | 单任务状态、顾问、渠道、错误、重试次数和抽样送审 |
| 顾问管理 | Twenty | 属性层级、列表详情关系 | 完整 CRM 导航和漏斗 | 顾问画像与门店属性可保存、刷新和重置 |
| 设置/治理 | shadcn 交互原则 | Dialog、Tabs、Toast、Focus、disabled | 通用 shadcn Dashboard | 连接器、审计和 Demo 场景集中管理，主要业务不被技术状态污染 |
| 试驾预约 | Cal.com / Cal.diy | 时间、地点、参与人、确认和状态 | 点击即写入固定记录 | 可编辑时间、地点、物品和备注，后端确定真实负责顾问 |
| 热点与洞察 | Plane + Twenty | 队列、证据详情和影响对象 | 无解释热度分、装饰图表 | 左队列、中证据、右影响与动作；所有趋势明确来源 |
| 知识中心 | Twenty | 属性层级、版本详情和关联记录 | 文档库模板照搬 | 左目录、中正文/diff、右影响和审核状态 |
| 客户 360 | Twenty + Chatwoot | 联系人属性、沟通历史、行动上下文 | 黑盒总分和销售漏斗 | 每个状态维度展示判断依据和下一最佳行动 |
| 质量与辅导 | Chatwoot + Plane | 待处理队列、原始沟通、指派和状态 | 员工排行榜、自动处罚 | 左信号、中证据、右员工说明与经理决定 |
| 全局视觉 | 乐道官方网页气质 | 明亮、温和、克制留白、服务导向 | 模仿官网组件或虚构品牌 Token | 延续浅色、清晰绿色、14px 以上正文和投屏可读性 |
