# 企业产品地图

## 产品定位

蔚见连接总部知识、门店经理、一线顾问和客户沟通，重点解决“信息变化是否及时传达到内容和客户”“顾问是否能持续推进下一步”“质量信号是否有证据并得到人工处理”。

## 顾问空间

| 页面 | 核心问题 | 关键输出 |
|---|---|---|
| 今日机会 | 今天先联系谁、为什么 | 优先机会和推荐动作 |
| 客户沟通 | 客户刚说了什么、下一步怎么做 | 会话、回复、试驾和记忆 |
| 内容作战台 | 如何生成可信且个性化的沟通 | 三平台内容、事实、合规、审核 |
| 跟进与承诺 | 我答应客户什么、何时完成 | 承诺状态、提醒和履约记录 |
| 客户档案 | 客户当前处于什么状态 | 客户 360、判断依据和下一最佳行动 |

## 门店经理空间

| 页面 | 核心问题 | 关键输出 |
|---|---|---|
| 门店雷达 | 门店现在有哪些需处理信号 | 机会、承诺、质量和风险队列 |
| 审核队列 | 内容能否发布 | 逐句证据、风险和人工决定 |
| 客户风险 | 哪些客户需要经理介入 | 证据、影响、责任人和截止时间 |
| 质量与辅导 | 哪些行为需要复核或辅导 | 员工说明、经理决定、辅导计划 |
| 承诺履约 | 哪些承诺即将到期或超时 | 提醒、协助和完成状态 |
| 活动执行 | 批量任务是否完成 | 单任务状态、失败重试和抽样审核 |

## 总部运营空间

| 页面 | 核心问题 | 关键输出 |
|---|---|---|
| 热点与洞察 | 哪些问题正在重复发生 | 证据、影响范围、负责人和任务 |
| 知识中心 | 哪条知识是当前有效版本 | 版本、diff、影响和重新核验 |
| 活动编排 | 如何规模化下发并监控 | 顾问/渠道任务明细和审核 |
| 优秀案例 | 哪些方法值得组织复用 | 匿名化案例和适用边界 |
| 效果验证 | 如何验证系统价值 | 实验设计、样本和未形成结论状态 |
| 系统治理 | 当前连接、场景和审计是什么 | Demo Adapter、审计、演示场景和边界 |

## 跨角色主线

1. 总部知识发生变化；
2. 系统创建版本和影响任务；
3. 顾问内容核验失效并重新核验；
4. 经理审核；
5. 顾问与客户沟通并形成承诺；
6. 质量信号出现时，员工补充上下文；
7. 经理确认辅导或沉淀优秀案例；
8. 结果反哺知识与下一次沟通。
