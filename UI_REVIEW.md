# UI Review v0.4.2

## 今日机会

改造前：筛选、队列和详情纵向占用较大，详情下半屏容易空白。改造后：360px 紧凑队列与弹性详情独立滚动，筛选和详情底部行动区固定；状态全部中文化。主操作为“开始处理”。

## 客户沟通

改造前：消息更像普通时间线，每条消息固定显示五个按钮。改造后：300px 会话列表、消息工作区和 320px 客户上下文组成三栏；客户、顾问和系统事件使用不同结构；每条消息只显示“转为承诺”，其他动作进入菜单；输入区固定在底部，右栏可折叠。

## 内容作战台

改造前：三栏存在但右栏默认堆满全部证据，编辑长内容后主操作离开视野。改造后：平台栏和底部命令栏固定；右栏先显示总体状态、风险和覆盖率，再按事实/风险 Tab 查看；正文变更时证据显示“待重新核验”；右栏可折叠。

## 跟进与承诺

改造前：确认、完成、延期和 Demo 操作同级排列。改造后：当前状态只保留一个主操作；修改、延期和经理协助进入更多菜单；模拟到期和超时进入独立演示工具。

## 客户档案

改造前：状态依据、隐私、时间线和下一行动同时展开。改造后：使用客户概览、状态判断、沟通时间线、承诺与任务、数据授权 Tabs；右栏下一最佳行动每项仅保留一个主操作，其他动作进入菜单；当前客户写入 URL 并支持刷新恢复。

## 顾问与门店

改造前：目录和编辑表单为两段式长页面。改造后：顾问目录、画像编辑、影响预览三栏；保存栏固定；右侧实时解释哪些字段参与生成并显示示例表达。

## 门店经理空间

门店雷达与质量辅导沿用“队列 → 证据 → 决策”。中栏优先展示原始业务上下文，系统判断作为辅助，右栏保留人工决策。不增加员工排名或 KPI 驾驶舱。

## 总部运营空间

热点与知识中心沿用“列表 → 详情/版本 → 影响与动作”。知识页面突出版本和影响对象，热点页面突出证据而非雷达图或词云。

## 响应式与可访问性

- 1366×768 无页面级横向溢出。
- 客户上下文和事实合规侧栏均可折叠。
- 图标按钮具有 aria-label；focus 状态可见；Dialog 锁定焦点。
- 支持 `prefers-reduced-motion`。
